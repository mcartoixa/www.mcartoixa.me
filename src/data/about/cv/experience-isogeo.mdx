I managed a small team of developers (SCRUM, Kanban).

I lead the design and development of Isogeo, a cataloguing platform for geospatial data centred on a RESTful API in JSON and comprised of various, mainly web, applications (<IconText name="devicon:visualstudio">_Visual Studio 2015_</IconText>, <IconText name="devicon:csharp">_C#_</IconText>, <IconText name="devicon:dot-net">_.NET 4.5_</IconText>, _ASP.NET WebApi 2_, _ASP.NET MVC 5_, _OWIN_, <IconText name="devicon:entityframeworkcore">_Entity Framework 6_</IconText>, <IconText name="devicon:microsoftsqlserver">_SQL Server 2016_</IconText>, <IconText name="devicon:javascript">_Javascript_</IconText>, <IconText name="devicon:nodejs">_node.js_</IconText>, <IconText name="devicon:backbonejs">_backbone.js_</IconText>, <IconText name="devicon:oauth">_OAuth 2_</IconText>). The platform was hosted on Windows Azure. I set up the continuous integration and delivery process for the project (_CruiseControl.NET_, _MSBuild_, <IconText name="devicon:grunt">_grunt.js_</IconText>, <IconText name="devicon:git">_Git_</IconText>, _Web Deploy_).

I designed and developed [GeoSIK](https://github.com/mcartoixa/GeoSIK), an open source library to develop and use OGC web services in .NET applications (<IconText name="devicon:visualstudio">_Visual Studio 2015_</IconText>, <IconText name="devicon:csharp">_C#_</IconText>, <IconText name="devicon:dot-net">_.NET 4.0_</IconText>, _LINQ_, <IconText name="devicon:microsoftsqlserver">_SQL Server 2014_</IconText>). I integrated GeoSIK into Isogeo to make it provide and consume standard CSW services ([Catalog Services for the Web](https://www.ogc.org/standards/cat/)).

In charge of the IT for the whole organization: I bought, set up and maintained servers (_DNS_, _DHCP_, _Active Directory_), workstations and laptops (<IconText name="devicon:windows8">_Windows Server 2012_</IconText>, <IconText name="devicon:windows8">_Windows 10_</IconText>).
